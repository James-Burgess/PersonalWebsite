# AGENTS.md — james-burgess/PersonalWebsite

## What this repo is

Personal website of James Burgess, deployed to Cloudflare Workers. The current live site is **v5** (Astro static site). Legacy iterations (v1–v4) are archived under `/archive/versionN/`.

## Directory map

| Path | What it is |
|------|------------|
| `v5/` | **Current site source** — Astro 5, Markdown-only content. |
| `static/` | Deployed static assets. **Do not edit directly** — generated by `deploy.sh`. |
| `server/` | Cloudflare Worker source (`server/src/index.js`). |
| `cv/` | Dynamic CV renderer (`cv/cv-index.js`) + JSON data files + templates. **Not wired into the main Worker entrypoint.** |
| `v1/`, `v2/` | Old Vue 2 + webpack sites. Build requires Docker (`node:14-alpine`). |
| `v3/`, `v4/` | Old static sites. `dist/` is already committed; no build step. |

## Build & deploy

### Full production build
```bash
./deploy.sh
```
This is the single source of truth. It:
1. Builds v1 and v2 via Docker (if `Dockerfile.build` exists).
2. Builds v5 with **Bun** (`bun install && bun run build`).
3. Rsyncs `v5/dist/` → `static/` (excluding `archive/`).
4. Copies v1–v4 `dist/` → `static/archive/versionN/`.
5. Rewrites absolute `/static/` refs to relative `static/` for v1/v2 `index.html`.

### v5 local dev
```bash
cd v5
bun install
bun run dev        # astro dev, port 3000, host: true
```
Use **Bun**, not npm, for v5. Astro config is in `v5/astro.config.mjs`.

### Worker local dev
```bash
cd server
npm install        # or npm ci
npm run dev        # wrangler dev
npm run deploy     # wrangler deploy
```

## Architecture notes

### v5 is Markdown + CSS only
- Every page section is a `.md` file in `v5/src/sections/`. Filenames are ordered (`01-tldr.md`, `02-experience.md`, …).
- **No frontmatter. No components in page stubs.** `index.astro` scans the directory, parses markdown with `gray-matter` + `marked`, and injects the HTML into `<section id="...">`.
- Build-time post-processing: paragraphs containing *only* inline `<code>` elements are transformed into `<div class="tech-tags">` with `<span class="tech-tag">` children. This is how tech pills are rendered.
- See `MARKDOWN-CSS.md` for the exact Markdown-to-CSS mapping conventions (job entries, repo cards, skill blocks, etc.).

### Worker routing (`server/src/index.js`)
- `/robots` — bot detection via User-Agent regex. Returns plain-text robot info to bots/search engines; returns an HTML page to humans.
- `/resume` — password-protected CV renderer (see CV system below).
- `/contact` — contact form. GET shows the form; POST sends an `ntfy.sh` alert to `jimmybcoza` and renders a success/error message.
- Everything else falls through to `env.ASSETS.fetch(request)` (static assets served from `static/` per `wrangler.toml`).
- `wrangler.toml` points `main = "server/src/index.js"` and assets `directory = "./static"`.

### CV system (`cv/`)
- Wired into the main Worker at **`/resume`** (`server/src/index.js`).
- Renders password-protected CVs from JSON data files in `cv/data/`.
- Route pattern: `/resume?data=marine-conservation&password=secret`. Falls back to `cv-default.json` if file or password is missing/wrong.
- `cv/template.js` exports `renderCV(data)` and `renderPasswordForm(dataFile, error)`.
- CV data JSON files are **bundled at build time** via static imports (`import ... from "../../cv/data/..."`). They are **never copied to `static/`** and cannot be accessed directly as raw JSON.

## Important constraints

- Do **not** edit `static/index.html` or `static/styles/global.css` directly. They are build artifacts from `v5/`. Edit source in `v5/src/` or `v5/public/` and run `deploy.sh`.
- v1 and v2 require Docker because they need Node 14. Do not try to build them with a modern Node version.
- No tests, no lint config, no typecheck, no CI pipelines in this repo.
